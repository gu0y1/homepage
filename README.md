# Academic Personal Homepage

A minimal, folder-based academic homepage for GitHub Pages.
Built with plain HTML + CSS + a small JS loader. Math rendered via [KaTeX](https://katex.org/).
Supports about, news, research, publications, projects, teaching, and contact pages.

## How to Add a New Item

### 1. Create a folder

```
publications/my-new-paper/
```

Use the same pattern for other sections:

```
projects/my-project/
research/my-research-topic/
teaching/my-course/
news/my-news-entry/
```

### 2. Add `meta.json`

**Publication:**
```json
{
  "title": "Paper Title",
  "authors": "Author A, <u>Your Name</u>, Author B",
  "venue": "ICML",
  "year": 2025,
  "summary": "One-sentence description.",
  "tags": ["topic"],
  "links": {
    "paper": "https://arxiv.org/abs/...",
    "code": "https://github.com/..."
  }
}
```

**Research project:**
```json
{
  "title": "Project Name",
  "year": 2024,
  "status": "Ongoing",
  "summary": "Brief description.",
  "tags": ["topic A", "topic B"]
}
```

**Project:**
```json
{
  "title": "Project Title",
  "year": 2026,
  "status": "Active",
  "summary": "Short project description.",
  "thumbnail": "thumbnail.jpg",
  "tags": ["tooling", "research"],
  "links": {
    "demo": "https://example.com/",
    "code": "https://github.com/..."
  }
}
```

**Teaching course:**
```json
{
  "title": "CS 101 — Intro to CS",
  "semester": "Spring 2025",
  "role": "Instructor",
  "year": 2025,
  "summary": "Course description."
}
```

**News entry:**
```json
{
  "title": "One paper accepted at ICML 2025",
  "date": "2025-03-01",
  "summary": "Short news description.",
  "links": {
    "publication": "publications/example-paper-icml2025/"
  }
}
```

### 3. (Optional) Add more files

- `thumbnail.jpg` / `thumbnail.png` / `thumbnail.svg` — auto-detected and shown in listing
- `index.html` — detail page (copy from an existing example)

Items may have their own detail page or not:

- If an item folder contains `index.html`, the title links to that page.
- If it does not, the title stays as text and any configured `links` are still shown.
- If a metadata item has `url` or `page_url`, the title links there instead.
- If a metadata item has `page`, it can be `true` for the folder page or a string such as `"details.html"`.

Useful shared fields:

- `title`: display title.
- `date`: used for news timeline sorting.
- `year`: used for sorting; publications also show it as a visible year badge.
- `summary`, `brief`, or `description`: short text shown in listings.
- `thumbnail`: optional path or URL for the listing image.
- `links`: either an object (`{"paper": "..."}`) or an array (`[{"label": "demo", "href": "..."}]`).

News is loaded automatically:

- The homepage shows the first 5 entries from `news/items.json`.
- `/news/` shows the full list as a timeline.
- Link paths in news metadata are written relative to the site root, such as `publications/example-paper-icml2025/`.

Detail pages one level below a section should put their title in the breadcrumb instead of repeating it in the page body:

```html
<script>
  initLayout('../../', {
    active: 'projects',
    breadcrumb: {
      sectionLabel: 'Projects',
      sectionHref: 'projects/',
      title: 'Project Title'
    }
  });
</script>
```

### 4. Rebuild

```bash
python build.py
```

This scans all folders, reads `meta.json`, and regenerates `items.json`.

If you set up the GitHub Action (`.github/workflows/build.yml`), this runs automatically on every push.

## Local Preview

Serve the site from the repository root:

```bash
python -m http.server 8000 --bind 127.0.0.1
```

Then open `http://127.0.0.1:8000/`.

Use a local server instead of opening `index.html` directly, because the item lists are loaded with `fetch()` from `items.json`.

## Math Support

KaTeX is loaded on every page. Use:

- Inline: `$E = mc^2$`
- Display: `$$\int_0^\infty e^{-x}\,dx = 1$$`

## Deploy to GitHub Pages

The repository includes a GitHub Actions workflow at `.github/workflows/build.yml`.

On every push to `main`, the workflow:

1. Checks out the repository.
2. Runs `python build.py` to regenerate `items.json`.
3. Uploads the static site as a GitHub Pages artifact.
4. Deploys the artifact to the `github-pages` environment.

You can also run the deployment manually from **Actions → Deploy site to GitHub Pages → Run workflow**.

For a project repository, the site URL is:

```
https://<username>.github.io/<repository-name>/
```

For this repository:

```
https://gu0y1.github.io/minimal-academic-page/
```
